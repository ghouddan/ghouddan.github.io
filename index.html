The Problem
Organizations need reliable methods to detect common Active Directory attack techniques before they become security incidents.
The Process
Deployed a full virtualized enterprise environment — Windows Server AD/DC, domain-joined clients, a Kali attacker VM, Elastic SIEM, Zeek, Suricata and Sysmon telemetry — then ran 10+ attack scenarios (Kerberoasting, Pass-the-Hash, credential dumping, lateral movement, persistence, C2) and wrote a Sigma rule for each, mapped to MITRE ATT&CK.
Tools Used
Elastic SIEMZeekSuricataSysmonSigmaMITRE ATT&CKKali Linux
Result / Impact
10+ documented attack-and-detect cycles published to GitHub with incident reports, IOC lists and detection-gap analysis, written to SOC reporting standards.

Roughly 287 new CVEs surface daily — manually triaging which ones actually matter isn't sustainable for a small team.
The Process
Built a pipeline pulling from NVD, CISA KEV and EPSS, ran everything through a 3-tier intelligent filter, then fed structured payloads to Gemini Flash to draft analyst-style briefings, delivered daily via Telegram on a cron schedule.
Tools Used
PythonNVD APICISA KEVEPSSGemini FlashTelegram Bot API
